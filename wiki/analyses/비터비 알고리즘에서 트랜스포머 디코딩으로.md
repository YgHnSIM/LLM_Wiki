# 비터비 알고리즘에서 트랜스포머 디코딩으로

[[비터비 알고리즘]]은 [[대규모 언어 모델]]의 직접적 조상이라기보다, 시퀀스 후보 공간을 어떻게 효율적으로 탐색할 것인가라는 문제의 고전적 기준점이다. 이 알고리즘은 [[은닉 마르코프 모델]]에서 가장 가능성 높은 상태열을 찾는 문제를 [[동적 계획법]]으로 해결했다.

[[N그램 모델]]과 HMM은 모두 [[마르코프 가정]]을 사용해 언어와 시퀀스를 계산 가능한 형태로 단순화했다. N그램은 다음 단어 예측을 짧은 문맥으로 제한했고, HMM은 은닉 상태 전이를 직전 상태에 의존하도록 제한했다. 이 단순화는 장거리 의존성을 잃는 대가로 현실적인 계산 가능성을 얻는 방식이었다.

비터비 알고리즘의 역사적 중요성은 완전 탐색 없이도 주어진 모델 아래의 전역 최적 경로를 찾을 수 있음을 보여 준 데 있다. 이는 [[음성 인식]], [[품사 태깅]], [[개체명 인식]]이 규칙 기반 추론에서 확률적 시퀀스 모델로 이동하는 데 중요한 기반이 되었다.

[[기계번역]]에서도 같은 문제가 나타났다. 번역은 단어 하나를 고르는 문제가 아니라 가능한 번역문 시퀀스, 구문 정렬, 단어 정렬 중 좋은 경로를 찾는 문제다. 통계적 기계번역은 여러 후보를 점수화하고 탐색하는 방식으로 이 문제를 다뤘고, 이후 [[신경망 기계번역]]은 문장 표현과 디코딩을 신경망 안으로 통합했다.

현대 트랜스포머는 HMM처럼 명시적 상태 전이표를 사용하지 않는다. 어텐션은 과거의 어느 위치가 중요한지 학습하고, 자기회귀 생성은 이전 토큰들을 조건으로 다음 토큰을 고른다. 따라서 트랜스포머 디코딩을 비터비 알고리즘의 단순 확장으로 보는 것은 부정확하다.

그러나 [[빔 탐색]]은 비터비와 중요한 철학을 공유한다. 가능한 모든 문장 후보를 추적할 수 없기 때문에, 각 단계에서 유망한 후보 일부만 남기고 나머지를 버린다. 차이는 비터비가 특정 모델 구조 아래에서 전역 최적성을 보장하는 반면, 빔 탐색은 보통 계산 가능한 근사 탐색이라는 점이다.

이 계보의 핵심은 모델이 바뀌어도 시퀀스 디코딩 문제가 사라지지 않는다는 것이다. HMM, CRF, 통계적 기계번역, 신경망 번역, LLM은 서로 다른 표현과 학습 방식을 쓰지만, 모두 거대한 후보 공간에서 그럴듯하고 유용한 출력 시퀀스를 선택해야 한다.
