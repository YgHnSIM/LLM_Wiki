# 대규모 언어 모델

대규모 언어 모델(Large Language Model, LLM)은 대량의 텍스트로 학습해 다음 토큰 예측, 대화, 요약, 번역, 코딩, 추론 보조 등을 수행하는 언어 모델이다. [[N그램 모델]]과 마찬가지로 주어진 문맥에서 다음 항목의 확률 분포를 추정한다는 문제 설정을 공유하지만, 단순한 카운트 테이블 대신 신경망과 [[분산 표현]]을 사용한다.

[[튜링 테스트]]의 관점에서 LLM은 언어 AI의 역사적 목표가 상당 부분 현실화된 사례다. GPT-4, Claude, Gemini 같은 모델은 제한된 조건의 대화에서 인간 응답과 구별하기 어려운 수준의 언어 행동을 보일 수 있다.

그러나 LLM의 유창성은 곧바로 진정한 이해나 의식을 증명하지 않는다. [[중국어 방 논증]]이 제기하는 것처럼, 적절한 언어 출력과 의미 이해 사이에는 여전히 철학적·기술적 간극이 있다.

보강 참고 자료의 관점에서 이 간극은 [[강한 AI]]와 [[약한 AI]]의 구분으로 정리된다. LLM을 마음 연구와 작업 자동화의 강력한 도구로 보는 것과, LLM 자체가 고유한 [[의도성]]을 가진다고 보는 것은 서로 다른 주장이다. 특히 [[기호 접지 문제]]는 모델의 기호 처리와 세계 지시가 어떻게 연결되는지 별도로 묻는다.

현대 LLM 평가는 인간다움만으로 충분하지 않다. 사실성, 환각, 편향, 도구 사용, 장기 작업 수행, 안전성, 사용자 과신 가능성 같은 기준이 함께 필요하다.

[[ELIZA]]의 관점에서 LLM의 유창성은 [[ELIZA 효과]]를 더 강하게 만들 수 있다. 사용자는 자연스러운 말투, 공감 표현, 기억처럼 보이는 반응을 실제 이해나 책임 있는 판단으로 오해할 수 있으므로, 특히 상담·교육·의료 같은 영역에서는 역할 한계와 안전한 전환 기준이 필요하다.

기계번역의 관점에서 LLM은 번역을 독립 도구가 아니라 대화형 작업 흐름의 일부로 통합한다. 번역, 요약, 용어 정리, 문체 조정, 문서 비교, [[후편집]]이 하나의 인터페이스 안에서 결합될 수 있다.

신경망의 관점에서 LLM은 [[퍼셉트론]]의 기본 아이디어를 거대한 규모로 확장한 후손이기도 하다. 내부의 선형층은 입력 표현에 [[가중치]] 행렬과 [[편향]]을 적용하며, 전체 모델은 오류를 줄이도록 매개변수를 반복적으로 조정한다.

촘스키의 관점에서 LLM은 흥미로운 긴장을 만든다. 명시적 [[변형생성문법]]이나 [[보편문법]]을 내장하지 않아도 문법적 출력을 만들지만, 내부 표현이 [[통사 구조]]와 [[재귀]]를 얼마나 안정적으로 학습하는지는 여전히 중요한 질문이다.

MADALINE의 관점에서 LLM 기반 음성 인터페이스는 언어 모델 하나로 완성되지 않는다. 실제 음성 제품은 [[적응 필터]], [[에코 제거]], [[음성 활동 감지]] 같은 신호 처리 전단과 결합되어야 하며, 입력 품질이 뒤의 언어 이해와 응답 생성 품질에 영향을 준다.

[[비터비 알고리즘]]의 관점에서 LLM 생성은 넓은 의미의 [[시퀀스 디코딩]] 문제다. 현대 트랜스포머는 HMM처럼 비터비 디코딩을 직접 중심에 두지는 않지만, [[빔 탐색]], 샘플링, 제약 디코딩처럼 후보 시퀀스를 선택하는 전략은 비터비가 보여 준 경로 탐색과 가지치기 문제의 현대적 변형이다.
