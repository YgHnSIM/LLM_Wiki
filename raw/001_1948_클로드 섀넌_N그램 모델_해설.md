# 해설: 1948 클로드 섀넌의 N그램 모델

## 1. 이 글을 읽기 전에 알아야 할 것

이 글은 N그램 모델 자체만 설명하는 글이 아니라, 자연어 처리가 왜 "규칙을 손으로 쓰는 방식"에서 "데이터에서 패턴을 배우는 방식"으로 이동했는지를 보여 주는 역사적 안내문이다. 핵심은 단순하다. 언어는 완전히 자유로운 기호 나열이 아니라, 앞에 나온 단어가 뒤에 올 단어의 가능성을 강하게 제한하는 통계적 구조를 가진다.

필수 선수 지식은 많지 않다. 다만 다음 개념을 알고 읽으면 글의 중심이 훨씬 선명해진다.

| 용어 | 설명 |
| --- | --- |
| N그램 (n-gram) | 텍스트에서 연속된 n개 항목으로 이루어진 묶음. 항목은 단어, 문자, 음절 등이 될 수 있다. |
| 유니그램 (unigram) | 하나의 항목만 보는 N그램. 단어 기반이면 단어 하나다. |
| 바이그램 (bigram) | 연속된 두 항목을 보는 N그램. 예: "peanut butter". |
| 트라이그램 (trigram) | 연속된 세 항목을 보는 N그램. 예: "peanut butter and". |
| 말뭉치 (corpus) | 모델이 통계를 배울 수 있도록 모아 둔 텍스트 데이터 집합. |
| 조건부 확률 (conditional probability) | 어떤 조건이 주어졌을 때 사건이 일어날 확률. N그램에서는 "앞 단어들이 주어졌을 때 다음 단어가 나올 확률"이다. |
| 마르코프 가정 (Markov assumption) | 다음 상태가 전체 과거가 아니라 제한된 최근 문맥에만 의존한다고 보는 가정. |
| 데이터 희소성 (data sparsity) | 가능한 조합은 엄청나게 많은데 실제 데이터에서 관찰된 조합은 적은 문제. |
| 스무딩 (smoothing) | 보지 못한 단어열에도 0이 아닌 적절한 확률을 주기 위해 확률 추정치를 조정하는 방법. |
| 백오프 (back-off) | 긴 문맥의 통계가 없으면 더 짧은 문맥의 통계로 물러나는 방법. |
| 퍼플렉서티 (perplexity) | 언어 모델이 다음 단어를 얼마나 덜 헷갈리며 예측하는지 나타내는 평가 지표. 낮을수록 보통 더 좋다. |

## 2. 배경 지식: 기초 → 중급 → 심화

### 기초: N그램은 "이어질 법함"을 세는 방법이다

N그램 모델은 단어의 의미를 이해하는 모델이 아니다. 이 모델은 텍스트에서 어떤 단어열이 얼마나 자주 나왔는지 세고, 그 빈도를 바탕으로 다음 단어의 가능성을 추정한다. 예를 들어 영어 말뭉치에서 "peanut butter and" 뒤에 "jelly"가 자주 나왔다면, 모델은 같은 문맥에서 "jelly"를 높은 확률의 후보로 둔다.

비유하면 N그램은 숙련된 독자의 감각을 아주 단순한 표로 만든 것이다. 사람은 "오늘 날씨가 참" 다음에 "좋다"가 올 가능성이 높다는 것을 경험적으로 안다. N그램 모델은 이런 감각을 "얼마나 자주 같이 나왔는가"라는 수치로 바꾼다.

### 중급: 마르코프 가정은 계산을 가능하게 만든다

언어 전체의 과거 문맥을 모두 고려하면 가능한 경우의 수가 폭발한다. 그래서 N그램 모델은 과감한 단순화를 한다. 다음 단어를 예측할 때 바로 앞의 한 단어, 두 단어, 또는 몇 단어만 본다. 이것이 본문에서 말하는 마르코프 가정이다.

이 가정은 현실을 완전히 반영하지 않는다. 긴 문장에서는 멀리 떨어진 주어, 목적어, 주제, 문체가 다음 단어 선택에 영향을 준다. 그러나 이 단순화 덕분에 모델을 실제로 학습하고 사용할 수 있다. N그램 모델의 강점과 약점은 모두 여기서 나온다. 계산은 쉬워지지만, 장거리 의미 관계는 놓친다.

### 심화: 정보 이론에서 언어 모델링으로 이어진 길

Shannon의 본래 관심사는 자연어 처리라기보다 통신이었다. 잡음이 있는 채널을 통해 메시지를 얼마나 효율적으로 보낼 수 있는가, 정보량을 어떻게 수학적으로 정의할 수 있는가가 중심 문제였다. 그런데 이 과정에서 언어가 매우 예측 가능한 구조를 가진다는 점이 드러났다.

이 관점은 이후 자연어 처리의 방향을 바꿨다. 언어를 일일이 규칙으로 설명하기보다, 실제 사용 데이터에서 패턴을 학습하는 접근이 가능해진 것이다. N그램 모델은 오늘날의 대규모 언어 모델과 구조는 다르지만, "주어진 문맥에서 다음 항목의 확률 분포를 추정한다"는 문제 설정을 공유한다.

## 3. 숲 보기: 글 전체의 구조와 핵심 통찰

본문은 다섯 흐름으로 전개된다.

첫째, Shannon의 1948년 연구가 정보 이론의 맥락에서 출발했지만 자연어 처리의 중요한 씨앗이 되었다는 점을 설명한다. 여기서 중요한 전환은 언어를 규칙의 체계가 아니라 통계적 현상으로 보는 것이다.

둘째, N그램의 기본 구조를 설명한다. 유니그램, 바이그램, 트라이그램처럼 연속된 단어 묶음을 세는 방식이 어떻게 언어의 국소적 패턴을 포착하는지 보여 준다.

셋째, N그램 모델이 예측에 어떻게 쓰이는지 설명한다. 앞선 단어열을 보고 말뭉치에서 비슷한 사례를 찾아 다음 단어의 확률을 추정한다는 점이 핵심이다.

넷째, 데이터 희소성 문제를 다룬다. N그램 길이가 길어질수록 더 많은 문맥을 얻지만, 가능한 조합 수가 폭발해 대부분의 조합을 관찰하지 못한다. Katz back-off, Good-Turing smoothing, Kneser-Ney smoothing은 이 문제를 완화하려는 대표적 방법이다.

다섯째, N그램의 한계가 신경망 언어 모델, 분산 표현, word2vec, GloVe, 트랜스포머로 이어지는 연구 의제를 만들었다고 설명한다. 본문의 가장 큰 통찰은 단순한 카운트 기반 모델이 실패했기 때문에 중요하지 않은 것이 아니라, 그 실패 지점이 다음 세대 모델의 방향을 정했다는 데 있다.

## 4. 나무 보기: 섹션별 상세 해설

### 도입부와 "1948: Shannon's N-gram Model"

이 부분은 Shannon의 논문을 통신 공학의 문제와 연결한다. 전화선과 전신 케이블은 언뜻 NLP와 멀어 보이지만, 두 영역 모두 불확실한 신호에서 정보를 추정한다는 공통점이 있다. 본문은 이 연결을 통해 N그램을 단순한 텍스트 처리 기법이 아니라 정보 이론적 사고의 산물로 위치시킨다.

주의할 점은 Shannon이 오늘날의 의미로 "N그램 언어 모델 제품"을 만든 것이 아니라는 점이다. 본문은 Shannon의 언어 통계 실험과 정보량 개념이 이후 N그램 모델링의 토대가 되었다는 역사적 연결을 설명한다.

### "The Anatomy of an N-gram"

이 섹션은 N그램의 단위를 설명한다. 핵심은 n이 커질수록 더 많은 문맥을 보지만, 그만큼 희귀한 조합이 많아진다는 것이다. "the quick brown fox jumps" 예시는 모델이 문장을 깊이 이해하지 않아도 연속된 단어 묶음을 추출할 수 있음을 보여 준다.

"peanut" 뒤에 "butter"가 더 자연스럽다는 예시는 문법과 통계의 차이를 드러낸다. "peanut giraffe"가 문법적으로 완전히 불가능한 것은 아니지만, 실제 언어 사용에서는 매우 드물다. N그램 모델은 이런 사용 빈도의 차이를 포착한다.

### "From Observation to Prediction"

이 섹션은 관찰된 빈도가 예측으로 바뀌는 과정을 설명한다. N그램 모델은 말뭉치에서 특정 문맥 뒤에 어떤 단어가 얼마나 자주 나왔는지 계산한다. 이 방식은 규칙 작성이 필요 없다는 장점이 있지만, 의미 이해가 없다는 한계도 분명하다.

마르코프 가정은 이 섹션의 핵심이다. 모델이 전체 문서를 기억하지 않고 최근 몇 단어만 보도록 제한하기 때문이다. 이 제한은 계산 가능성을 확보하는 장치이자, 긴 문맥을 잃어버리는 원인이다.

### "The Data Sparsity Problem and Its Solutions"

이 부분은 N그램 모델을 실제로 사용할 때 가장 중요한 문제를 다룬다. 가능한 단어 조합의 수는 너무 많아서, 아무리 큰 말뭉치라도 대부분의 조합을 포함하지 못한다. 관찰되지 않은 조합에 확률 0을 주면 모델은 새 문장을 거의 처리하지 못한다.

Katz back-off는 "긴 문맥을 못 봤으면 짧은 문맥으로 돌아가라"는 원칙이다. Good-Turing smoothing은 드물게 관찰된 사건을 이용해 아직 관찰되지 않은 사건의 질량을 추정한다. Kneser-Ney smoothing은 단어가 얼마나 많이 나왔는가뿐 아니라 얼마나 다양한 문맥에 나왔는가를 중시한다. 특히 Kneser-Ney의 직관은 중요하다. 어떤 단어가 한 가지 관용구에서만 많이 나오는지, 여러 환경에서 고르게 나오는지는 새로운 문맥에서의 가능성을 판단할 때 다르게 취급되어야 한다.

### "The Path Forward"

마지막 섹션은 N그램 모델의 한계를 신경망 언어 모델의 등장과 연결한다. N그램은 "car"와 "automobile"이 비슷하다는 사실을 알 수 없다. 두 단어는 서로 다른 표면 토큰이기 때문이다. 분산 표현은 이 문제를 벡터 공간으로 해결하려 했다. 의미가 비슷한 단어를 가까운 벡터로 놓으면, 표면형이 달라도 일반화가 가능해진다.

트랜스포머와 대규모 언어 모델은 N그램보다 훨씬 긴 문맥과 복잡한 관계를 처리한다. 그러나 다음 토큰을 예측한다는 기본 문제 설정은 여전히 Shannon의 통계적 언어관과 이어져 있다.

## 5. 본문 밖으로 확장하기

### 본문의 주장

본문의 주장은 N그램 모델이 단순한 과거 기술이 아니라 현대 언어 AI의 지적 출발점이라는 것이다. 핵심 연결고리는 세 가지다. 언어는 통계적 규칙성을 가진다. 다음 항목 예측은 언어 모델링의 중심 문제다. 데이터에서 패턴을 배우는 접근은 규칙 기반 접근보다 확장성이 크다.

### 외부 자료 기반 보충

Shannon의 "A Mathematical Theory of Communication"은 1948년 Bell System Technical Journal에 실린 논문으로, 정보량, 엔트로피, 통신 채널, 잡음, 부호화 문제를 수학적으로 다루었다. 이 논문 자체가 현대 정보 이론의 출발점으로 평가된다.

Katz의 1987년 논문은 희소 데이터에서 언어 모델 확률을 추정하는 대표적 백오프 접근을 제시했다. 이후 Kneser와 Ney의 1995년 논문은 백오프 분포를 더 정교하게 설계해 퍼플렉서티와 음성 인식 성능을 개선했다.

Bengio 등의 2003년 Neural Probabilistic Language Model은 단어의 분산 표현과 신경망 기반 다음 단어 예측을 연결했다. word2vec, GloVe, Transformer는 이 흐름을 더 확장하여 표면 단어열을 넘어 벡터 표현, 어텐션, 대규모 사전학습으로 이어졌다.

### 현재 기술 흐름에서의 평가

2026년 기준으로 주류 언어 모델은 대부분 트랜스포머 계열 대규모 모델이다. 이들은 N그램처럼 짧은 고정 창의 카운트 테이블을 조회하지 않고, 토큰 임베딩과 어텐션을 이용해 훨씬 넓은 문맥에서 다음 토큰 분포를 계산한다. 최근 흐름은 단순히 문맥 창을 늘리는 것뿐 아니라, 검색 증강 생성(RAG), 메모리 시스템, 도구 사용, 컨텍스트 엔지니어링처럼 모델 외부의 정보를 어떻게 넣고 관리할 것인가로 확장되고 있다.

그럼에도 N그램은 여전히 쓸모가 있다. 작은 데이터, 제한된 계산 자원, 빠른 기준선, 디버깅 가능한 확률 모델이 필요한 경우 N그램은 실용적이다. 검색, 철자 교정, OCR 후처리, 도메인 제한 자동 완성, 언어 모델 평가 교육용 예제에서는 여전히 좋은 출발점이다.

### 한계와 비판적 관점

N그램의 가장 큰 한계는 의미를 표현하지 못한다는 점이다. 같은 뜻의 단어라도 표면형이 다르면 완전히 다른 항목으로 취급한다. 또 긴 문맥을 보지 못하기 때문에 문서 전체의 주제, 화자의 의도, 앞 문장과 뒤 문장의 논리 관계를 다루기 어렵다.

그러나 신경망 모델이 이 한계를 모두 해결한 것은 아니다. 대규모 언어 모델도 여전히 다음 토큰 예측을 기반으로 하며, 사실성, 근거 추적, 긴 문맥에서의 정보 선택, 비용과 지연 시간 문제를 가진다. 따라서 N그램에서 출발한 질문, 즉 "문맥에서 다음 항목의 확률을 어떻게 추정할 것인가"는 여전히 살아 있다.

### 추천 학습 경로

초급자는 먼저 조건부 확률, 엔트로피, 유니그램/바이그램/트라이그램 예제를 손으로 계산해 보는 것이 좋다. 짧은 문장 몇 개로 직접 빈도표를 만들면 N그램 모델의 직관이 빠르게 잡힌다.

중급자는 스무딩을 공부해야 한다. Laplace smoothing, Good-Turing smoothing, Katz back-off, Kneser-Ney smoothing을 비교하면 "보지 못한 사건에 어떻게 확률을 줄 것인가"라는 언어 모델링의 핵심 난제가 보인다.

고급자는 N그램에서 신경망 언어 모델로 넘어가는 지점을 추적하면 좋다. Bengio의 Neural Probabilistic Language Model, word2vec, GloVe, Transformer를 순서대로 보면 표면 카운트에서 분산 표현과 어텐션으로 이어지는 변화가 선명해진다.

실무자는 N그램을 "낡은 모델"이 아니라 "빠르고 해석 가능한 기준선"으로 다루는 편이 좋다. 복잡한 대규모 모델을 쓰기 전에 N그램 기준선을 세우면 문제 자체가 국소 패턴으로 충분한지, 아니면 긴 문맥과 의미 표현이 필요한지 판단하기 쉬워진다.
