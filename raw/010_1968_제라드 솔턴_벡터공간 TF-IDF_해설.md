# 해설: 벡터 공간 모델과 TF-IDF: 현대 정보 검색과 의미 검색의 기반

## 1. 이 글을 읽기 전에 알아야 할 것

이 글은 정보 검색의 역사를 "문자열 일치"에서 "기하학적 유사성 계산"으로 바뀐 과정으로 설명한다. 핵심은 문서와 질의를 단어 목록이 아니라 벡터로 표현하면, 검색이 수학적 거리 또는 각도 계산 문제가 된다는 점이다.

읽기 전에 세 가지를 구분하면 좋다. 첫째, **검색**은 문서 컬렉션에서 사용자의 정보 요구에 맞는 문서를 찾는 문제다. 둘째, **랭킹**은 찾은 문서들을 관련성 높은 순서로 정렬하는 문제다. 셋째, **표현**은 문서와 질의를 컴퓨터가 계산할 수 있는 형태로 바꾸는 문제다. 벡터 공간 모델은 세 번째 문제를 풀면서 첫 번째와 두 번째 문제까지 바꾸었다.

TF-IDF는 "많이 나온 단어가 중요하다"는 생각과 "아무 데나 다 나오는 단어는 덜 중요하다"는 생각을 결합한다. 즉 한 문서 안에서는 자주 나오지만 전체 문서 집합에서는 드문 단어를 높게 평가한다. 이 원리는 단순하지만, 검색 시스템이 주제와 관련성을 계산하는 데 매우 강력한 출발점이 되었다.

다만 역사적 세부에는 약간의 주의가 필요하다. 원문은 1968년을 중심으로 설명하지만, 널리 인용되는 "A Vector Space Model for Automatic Indexing" 논문은 Salton, Wong, Yang의 1975년 CACM 논문이다. 1968년은 솔턴의 "Automatic Information Organization and Retrieval"과 SMART 프로젝트가 정보 검색의 수학적·자동화 접근을 본격화한 시기로 이해하는 것이 더 정확하다. 이 해설에서는 본문 흐름은 유지하되, 참고자료 섹션에서 이 역사적 맥락을 분리해 둔다.

## 2. 배경 지식: 기초 → 중급 → 심화

### 기초

**정보 검색**은 데이터베이스 질의와 다르다. 데이터베이스 질의는 조건을 정확히 만족하는 레코드를 찾는 경우가 많지만, 정보 검색은 사용자가 원하는 정보에 "얼마나 관련 있는지"를 다룬다. 사용자가 "딥러닝 학습 안정화"를 검색했을 때, 정확히 그 문구가 없는 문서도 관련 있을 수 있다.

**불리언 검색**은 AND, OR, NOT 같은 논리 연산자로 문서를 찾는 방식이다. 정확하고 설명 가능하지만, 결과가 너무 많거나 너무 적어지기 쉽다. 또한 관련성의 정도를 표현하기 어렵다.

**벡터**는 숫자의 목록이다. 문서를 벡터로 표현한다는 것은 문서의 내용을 여러 숫자로 바꾸는 것이다. TF-IDF 벡터에서는 각 차원이 단어 하나에 대응하고, 각 값은 그 단어가 해당 문서에서 얼마나 중요한지를 나타낸다.

**코사인 유사도**는 두 벡터가 같은 방향을 가리키는 정도를 측정한다. 문서 길이보다 단어 분포의 방향을 보므로, 긴 문서와 짧은 문서도 같은 주제를 다루면 비슷하게 평가할 수 있다.

### 중급

TF-IDF는 두 구성 요소로 나뉜다.

\[
\text{tfidf}(t, d) = tf(t, d) \times idf(t)
\]

\(tf(t,d)\)는 용어 \(t\)가 문서 \(d\) 안에 얼마나 자주 나타나는지 나타낸다. \(idf(t)\)는 그 용어가 전체 문서 집합에서 얼마나 드문지 나타낸다.

\[
\text{idf}(t) = \log\frac{N}{df(t)}
\]

\(N\)은 전체 문서 수이고, \(df(t)\)는 용어 \(t\)가 등장한 문서 수다. 용어가 모든 문서에 나타나면 \(N/df(t)\)는 1에 가까워지고 로그값은 낮다. 반대로 적은 문서에만 나타나면 IDF가 높아진다.

이 방식은 문서의 "구별되는 특징"을 찾는 데 유용하다. 예를 들어 AI 논문 집합에서 "model"은 너무 흔해서 변별력이 낮을 수 있지만, "backpropagation", "BM25", "contrastive learning" 같은 용어는 특정 주제를 강하게 가리킬 수 있다.

### 심화

벡터 공간 모델의 깊은 의미는 "의미를 공간에 배치한다"는 관점이다. 초기에는 각 차원이 단어 하나였기 때문에 희소하고 고차원적인 공간이었다. 현대 임베딩에서는 차원이 사람이 해석하기 어려운 학습된 특징이지만, 여전히 텍스트를 벡터로 놓고 유사성을 계산한다.

이 변화는 **희소 표현**에서 **조밀 표현**으로의 이동으로 볼 수 있다. TF-IDF는 대부분의 값이 0인 희소 벡터를 만든다. 반면 Word2Vec, BERT, dense retriever의 임베딩은 대부분의 차원이 실수 값을 갖는 조밀 벡터를 만든다. 희소 표현은 정확한 용어 매칭과 해석 가능성에 강하고, 조밀 표현은 동의어·문맥·의미적 유사성에 강하다.

2026년 기준의 검색 시스템은 두 방식을 함께 쓰는 경우가 많다. BM25 같은 희소 검색은 빠르고 희귀 용어에 강하며, dense retrieval은 의미적으로 비슷하지만 단어가 다른 문서를 찾는 데 유리하다. RAG 시스템에서도 BM25 후보 생성, dense retrieval, cross-encoder 재랭킹, LLM 응답 생성이 결합되는 구조가 흔하다.

### 용어 사전

| 원문 용어 | 한국어 표현 | 설명 |
|---|---|---|
| Vector Space Model, VSM | 벡터 공간 모델 | 문서와 질의를 벡터로 표현하고 기하학적 유사성으로 검색하는 모델 |
| TF-IDF | TF-IDF | 문서 내 빈도와 전체 컬렉션 내 희귀성을 결합한 용어 가중치 |
| Term Frequency, TF | 용어 빈도 | 특정 문서에서 용어가 얼마나 자주 나타나는지 |
| Inverse Document Frequency, IDF | 역문서 빈도 | 용어가 전체 문서 집합에서 얼마나 드문지 나타내는 값 |
| Cosine similarity | 코사인 유사도 | 두 벡터의 방향 유사성을 측정하는 척도 |
| Boolean retrieval | 불리언 검색 | AND, OR, NOT 같은 논리 연산으로 문서를 찾는 방식 |
| Relevance ranking | 관련성 순위화 | 문서를 관련성 높은 순서로 정렬하는 과정 |
| Sparse vector | 희소 벡터 | 대부분의 차원 값이 0인 벡터 |
| Dense vector | 조밀 벡터 | 대부분의 차원이 실수 값을 갖는 벡터 |
| Vocabulary mismatch | 어휘 불일치 | 같은 의미를 서로 다른 단어로 표현해 검색이 실패하는 문제 |
| Bag of words | 단어 주머니 | 어순과 문법 관계를 무시하고 단어 출현만 보는 표현 |
| BM25 | BM25 | TF-IDF 계열의 확률적 랭킹 함수로 현대 검색의 강력한 기준선 |
| Dense retrieval | 조밀 검색 | 신경망 임베딩으로 질의와 문서를 표현해 검색하는 방법 |
| Hybrid search | 하이브리드 검색 | 희소 검색과 조밀 검색을 결합하는 검색 방식 |

## 3. 숲 보기: 글 전체의 구조와 핵심 통찰

본문은 정보 검색의 발전을 세 단계로 설명한다. 먼저 불리언 검색의 한계를 제시한다. 정확히 일치하는 문서는 찾을 수 있지만, 관련성의 정도를 계산하거나 다른 표현을 쓰는 문서를 찾기 어렵다는 문제다. 다음으로 벡터 공간 모델과 TF-IDF를 소개한다. 문서와 질의를 벡터로 표현하고, 코사인 유사도로 관련성을 계산한다. 마지막으로 이 관점이 현대 임베딩, 의미 검색, 추천 시스템, 대규모 언어 모델의 벡터 연산까지 이어졌다고 설명한다.

핵심 통찰은 두 가지다. 첫째, 문서의 의미를 완벽히 이해하지 않아도 단어 분포를 잘 표현하면 유용한 관련성 계산이 가능하다. 둘째, 텍스트를 벡터 공간에 놓는 순간 검색은 기하학 문제가 된다. 이 관점은 이후 단어 임베딩, 문장 임베딩, dense retrieval, RAG까지 이어진다.

이 글의 중요한 균형은 TF-IDF를 과대평가하지 않는 데 있다. TF-IDF는 의미를 진짜로 이해하지는 않는다. 동의어, 다의어, 어순, 문맥을 잘 처리하지 못한다. 그러나 바로 그 한계가 이후 의미 임베딩과 신경망 검색의 동기가 되었다. 따라서 TF-IDF는 낡은 도구라기보다, 현대 의미 검색이 무엇을 계승하고 무엇을 보완했는지 이해하기 위한 기준점이다.

## 4. 나무 보기: 섹션별 상세 해설

### 1968년: 벡터 공간 모델과 TF-IDF

이 섹션은 정보 검색의 출발 문제를 규모의 문제로 제시한다. 문서가 적을 때는 사람이 색인하고 분류할 수 있지만, 문서가 폭증하면 수작업 분류는 비싸고 느리며 일관성도 떨어진다.

솔턴의 핵심 기여는 이 문제를 "표현" 문제로 본 것이다. 문서를 컴퓨터가 비교할 수 있는 수학적 객체로 바꾸면, 검색은 수작업 판단이 아니라 계산 가능한 문제가 된다. 이 사고방식은 현대 AI의 기본 태도와도 맞닿아 있다. 좋은 표현을 만들면 어려운 판단 문제를 벡터 연산으로 바꿀 수 있다.

### 문제: 키워드만으로는 충분하지 않을 때

불리언 검색의 한계는 "전부 아니면 전무"다. 조건을 만족하면 검색되고, 만족하지 않으면 제외된다. 하지만 실제 관련성은 연속적이다. 어떤 문서는 매우 관련 있고, 어떤 문서는 조금 관련 있으며, 어떤 문서는 거의 관련이 없다.

또 하나의 문제는 표현 차이다. 사용자가 쓰는 단어와 문서 작성자가 쓰는 단어가 다를 수 있다. "digital computer"와 "electronic computer"는 과거 맥락에서 비슷한 대상을 가리킬 수 있지만, 단순 문자열 매칭은 이를 알 수 없다.

이 섹션은 벡터 공간 모델이 왜 필요했는지 설명하는 역할을 한다. 검색은 단어 일치만으로는 부족하고, 관련성의 정도를 계산해야 한다.

### 돌파구: 의미의 기하학

이 부분은 글의 중심이다. 문서와 질의를 벡터로 만들면 "얼마나 관련 있는가"를 "두 벡터가 얼마나 같은 방향을 보는가"로 바꿀 수 있다.

각 차원이 단어이고, 각 값이 단어의 가중치라고 생각하면 된다. 비슷한 주제의 문서는 비슷한 단어를 비슷한 비율로 사용하므로 벡터 방향도 비슷해진다. 반대로 전혀 다른 주제의 문서는 서로 다른 단어 차원에서 값이 커지므로 벡터 방향이 달라진다.

여기서 "의미"라는 표현은 조심해서 읽어야 한다. TF-IDF 벡터가 인간처럼 의미를 이해한다는 뜻이 아니다. 단어 분포의 기하학이 의미적 관련성의 근사치로 작동한다는 뜻이다.

### 코사인 유사도

코사인 유사도는 문서 길이 문제를 완화한다. 단순 내적만 쓰면 긴 문서가 더 많은 단어를 갖기 때문에 높은 점수를 받기 쉽다. 코사인 유사도는 벡터 크기로 나누어 방향을 비교하므로, 길이가 다르더라도 주제가 비슷하면 높은 점수를 줄 수 있다.

다만 코사인 유사도도 완벽하지 않다. 두 문서가 같은 단어를 거의 공유하지 않으면, 의미가 비슷해도 낮은 점수가 나온다. 이 문제가 이후 어휘 불일치 문제와 dense embedding의 필요성으로 이어진다.

### TF-IDF

TF는 문서 내부의 중요도를 본다. 어떤 단어가 한 문서에서 많이 나오면 그 문서의 주제와 관련 있을 가능성이 높다. IDF는 컬렉션 전체에서의 변별력을 본다. 모든 문서에 다 나오는 단어는 관련성 판단에 별 도움이 되지 않는다.

TF-IDF의 장점은 지역 정보와 전역 정보를 동시에 본다는 점이다. 특정 문서에서는 중요하고 전체 컬렉션에서는 드문 단어가 높은 점수를 받는다. 이 원리는 현대 검색에서도 여전히 유효하다.

로그를 쓰는 이유는 극단값을 줄이기 위해서다. 아주 드문 단어가 지나치게 큰 영향을 주면 검색 결과가 불안정해질 수 있다. 로그는 값의 차이를 완만하게 만들어 더 안정적인 가중치를 만든다.

### 구현과 개선

이 섹션은 이론을 실제 시스템으로 옮길 때 생기는 문제를 다룬다. 문서가 수천 개이고 어휘가 수만 개이면 모든 문서를 거대한 벡터로 저장하는 것은 낭비가 크다. 대부분의 문서는 대부분의 단어를 포함하지 않기 때문이다.

희소 벡터 표현은 이 문제를 해결한다. 0이 아닌 값만 저장하면 메모리와 계산량을 크게 줄일 수 있다. 오늘날 검색 엔진의 역색인(inverted index)도 이 생각과 연결된다. 어떤 단어가 어떤 문서에 나타나는지 효율적으로 저장하고 검색한다.

관련성 피드백도 중요하다. 사용자가 어떤 결과가 관련 있다고 알려 주면, 시스템은 질의 벡터를 관련 문서 쪽으로 이동시킬 수 있다. 이는 사용자의 피드백으로 표현을 조정하는 초기 형태의 상호작용적 검색이다.

### 응용

본문은 벡터 공간 모델이 도서관 검색을 넘어 NLP 전반으로 확산된 과정을 설명한다. 문서 군집화, 텍스트 분류, 스팸 필터, 감성 분석은 모두 텍스트를 벡터 특징으로 바꾸면 기계 학습 문제로 처리할 수 있다.

특히 중요한 연결은 분포 의미론이다. 문서를 단어 벡터로 표현할 수 있다면, 단어도 그 단어가 등장하는 문맥으로 벡터화할 수 있다. 이 생각은 Word2Vec 같은 현대 단어 임베딩으로 이어진다.

검색 엔진에서 TF-IDF가 계속 쓰인다는 점도 실무적으로 중요하다. 최신 시스템이 모두 신경망만 쓰는 것은 아니다. 빠른 후보 생성과 정확한 키워드 매칭에는 여전히 희소 검색이 강하다.

### 한계

벡터 공간 모델의 가장 큰 약점은 단어를 독립된 차원으로 본다는 점이다. "car"와 "automobile"은 의미상 가깝지만, TF-IDF에서는 서로 다른 차원이다. 두 단어가 직접 겹치지 않으면 유사성이 낮게 계산된다.

다의어도 문제다. "bank"는 금융기관과 강둑을 모두 뜻할 수 있지만, 하나의 차원으로 합쳐진다. 따라서 실제로는 다른 주제의 문서가 같은 단어 때문에 비슷하게 보일 수 있다.

어순과 구문도 사라진다. 단어 주머니 표현은 어떤 단어가 나왔는지는 보지만, 누가 누구에게 무엇을 했는지는 보지 못한다. 정보 검색에서는 어느 정도 충분할 수 있지만, 질의응답이나 번역 같은 깊은 언어 이해에는 부족하다.

### 어휘 불일치 문제

어휘 불일치는 검색에서 매우 실무적인 문제다. 사용자는 "의사"를 검색하지만 문서는 "전문의"나 "의료진"이라고 쓸 수 있다. 단어가 다르면 희소 검색은 놓치기 쉽다.

이 문제를 해결하기 위해 질의 확장, 동의어 사전, 시소러스, 잠재 의미 분석, 임베딩 기반 검색이 발전했다. 현대 dense retrieval은 단어가 달라도 의미가 비슷한 문서를 찾기 위해 학습된 벡터 공간을 사용한다.

### 유산

이 섹션은 벡터 공간 모델을 현대 언어 AI의 조상으로 해석한다. 오늘날의 임베딩은 TF-IDF처럼 단어 차원에 직접 대응하지는 않지만, 텍스트를 벡터로 놓고 유사성을 계산한다는 기본 틀은 같다.

어텐션과의 연결은 개념적 연결이다. 트랜스포머의 query, key, value는 TF-IDF의 질의 벡터와 문서 벡터와 같은 것은 아니지만, 벡터 간 유사성 계산을 통해 관련 정보를 선택한다는 점에서 같은 기하학적 사고를 공유한다.

### 현대 검색과 검색 기반 시스템

현대 검색은 고전적 희소 검색과 신경망 기반 조밀 검색을 결합하는 방향으로 발전했다. BM25는 TF-IDF의 후손으로 볼 수 있는 강력한 희소 랭킹 함수다. Dense retrieval은 문장이나 문단을 임베딩 벡터로 바꾸어 의미적으로 가까운 문서를 찾는다.

RAG 시스템에서는 이 두 방식의 결합이 특히 중요하다. 희소 검색은 정확한 이름, 코드, 희귀 용어를 잘 찾고, 조밀 검색은 표현이 달라도 의미가 가까운 문서를 잘 찾는다. 둘을 결합한 뒤 cross-encoder나 LLM으로 재랭킹하면 품질이 좋아지는 경우가 많다.

### 인접 분야와 결론

벡터 공간 모델은 문서뿐 아니라 여러 특징으로 표현되는 거의 모든 객체에 적용할 수 있다. 유전자, 이미지, 음악, 논문 인용 패턴도 벡터로 표현하면 유사성을 계산할 수 있다. 이것이 이 모델의 강력한 일반성이다.

결론의 핵심은 벡터 공간 모델과 TF-IDF가 단순한 과거 기술이 아니라 "의미를 벡터 공간에서 계산한다"는 패러다임의 출발점이라는 것이다. 현대 시스템은 훨씬 복잡해졌지만, 그 기본 직관은 여전히 살아 있다.

## 5. 본문 밖으로 확장하기

> 본문의 주장: 벡터 공간 모델은 문서와 질의를 벡터로 표현해 관련성 계산을 기하학 문제로 바꾸었다.

외부 자료 기반 보충: Salton, Wong, Yang의 1975년 CACM 논문 "A Vector Space Model for Automatic Indexing"은 문서 검색과 패턴 매칭 환경에서 저장된 개체와 검색 요청을 벡터 공간에서 비교하는 접근을 제시했다. 솔턴의 1968년 저서 "Automatic Information Organization and Retrieval"은 자동 정보 조직과 검색의 수학적·통계적 접근을 폭넓게 정리했다.

현재 기술 흐름에서의 평가: 2026년 현재 검색 시스템은 단순 TF-IDF만 쓰지는 않지만, 텍스트를 벡터화하고 유사도 계산으로 검색한다는 기본 구조는 dense retrieval, vector database, RAG에서 계속 핵심이다.

> 본문의 주장: TF-IDF는 문서 안에서 자주 나오고 전체 컬렉션에서는 드문 단어를 중요하게 본다.

외부 자료 기반 보충: Manning, Raghavan, Schütze의 "Introduction to Information Retrieval"은 term weighting, vector space model, ranked retrieval을 체계적으로 설명한다. 이 교재는 TF-IDF를 현대 정보 검색의 기본 가중치 방식으로 다룬다.

현재 기술 흐름에서의 평가: BM25는 TF-IDF식 직관을 문서 길이 정규화와 용어 빈도 포화로 개선한 랭킹 함수로, 여전히 Elasticsearch, Lucene 계열 검색과 RAG 후보 검색에서 강한 기준선이다.

> 본문의 주장: TF-IDF의 한계는 동의어, 다의어, 문맥, 어순을 처리하지 못한다는 데 있다.

외부 자료 기반 보충: Dense Passage Retrieval은 질의와 문단을 신경망이 학습한 조밀 벡터로 표현해 open-domain QA에서 관련 문단을 검색하는 접근을 제시했다. SPLADE 같은 learned sparse retrieval은 희소 표현의 해석 가능성과 신경망 기반 의미 확장을 결합하려 한다.

현재 기술 흐름에서의 평가: 조밀 임베딩은 어휘 불일치를 줄이지만, 희귀 고유명사나 정확한 문자열 검색에서는 BM25보다 약할 수 있다. 그래서 2024-2026년의 실무 검색과 RAG에서는 hybrid search가 표준적 선택지로 자리 잡고 있다.

> 본문의 주장: 벡터 공간 모델의 영향은 NLP를 넘어 추천, 비전, 생물정보학, 과학계량학까지 확장되었다.

외부 자료 기반 보충: 벡터 공간 방법은 여러 특징으로 표현되는 객체 간 유사성을 계산하는 일반 프레임워크다. 문서의 용어 빈도, 이미지의 시각 단어, 사용자의 구매 이력, 논문의 인용 패턴은 모두 벡터로 표현될 수 있다.

현재 기술 흐름에서의 평가: 현대 AI의 많은 시스템은 "모든 것을 임베딩으로 만들고 가까운 것을 찾는다"는 구조를 갖는다. 이는 강력하지만, 벡터 유사성이 항상 인간의 의미 판단과 일치하지는 않는다. 평가 데이터, 도메인 적합성, 재랭킹, 설명 가능성이 함께 필요하다.

### 추천 학습 경로

초급자는 작은 문서 3개와 질의 1개를 직접 만들어 TF, IDF, TF-IDF, 코사인 유사도를 손으로 계산해 보는 것이 좋다. 이 과정을 한 번 해 보면 벡터 공간 모델의 직관이 바로 잡힌다.

중급자는 Manning et al.의 "Introduction to Information Retrieval"에서 ranked retrieval, vector space model, evaluation, relevance feedback을 읽고, scikit-learn의 `TfidfVectorizer`로 문서 분류나 검색 실험을 해 보면 좋다.

고급자는 BM25, dense retrieval, hybrid retrieval, cross-encoder reranking, RAG 평가로 확장하면 된다. 이 단계의 핵심 질문은 "정확한 용어 매칭과 의미적 유사성 검색을 어떤 비율로 결합할 것인가"이다.
